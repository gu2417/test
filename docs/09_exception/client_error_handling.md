# 클라이언트 에러 처리

## 1. 연결 끊김 감지

- recv thread 가 `recv()=0` 또는 오류 → `state = DISCONNECTED`.
- send thread 도 깨워 `write` 실패 시 조용히 종료.
- UI 는 상단에 빨간 배너 "서버 연결 끊김 — 재연결 중...".

## 2. 재연결 정책

- 지수 백오프: 1s → 2s → 4s → 8s (상한). 최대 5회.
- 실패 시 모달 "재연결 실패. Enter 로 재시도, q 로 종료."
- 재연결 성공 시:
  - 기존 인증 토큰이 없으므로 **자동 재로그인 시도** — 저장된 비밀번호 없으므로 LOGIN 화면으로 복귀.
  - 이전 방/DM 컨텍스트는 메인화면에서 재선택.

## 3. 렌더 실패

- `write(1, ...)` 가 `EAGAIN` → 짧게 sleep 후 재시도. 5회 실패 시 전체 재그리기.
- 터미널 크기 < 80×24 → "터미널 크기를 80×24 이상으로 조정해 주세요" 전면 메시지.
- `SIGWINCH` 수신 → 레이아웃 재계산 + full redraw.

## 4. 사용자 입력 에러

- 슬래시 커맨드 파싱 실패 → 하단 status bar 에 빨간 텍스트 "알 수 없는 명령: /foo".
- 서버가 `*_RES|<code>` 반환 → `08_api/error_codes.md` 매핑으로 한글 메시지 표시(2~3초 후 자동 소멸).

## 5. 전역 guard

- `SIGINT` (Ctrl-C) → graceful: `LOGOUT` 전송 + fd close + 터미널 raw 모드 복원.
- `SIGTERM` → 동일.
- 터미널 raw 모드 복원은 `atexit` 로도 등록(세그폴트 시에도 터미널이 깨지지 않도록 최소 보장).
