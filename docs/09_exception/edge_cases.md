# 엣지 케이스 카탈로그

## 인증

- **중복 로그인**: 이미 접속 중인 id 로 LOGIN → 기존 세션 kick 하지 않고 신규를 `STATE_CONFLICT`(6) 로 reject. (v2.1 에서 "기존 세션 강제 종료 옵션" 고려)
- **회원가입 도중 DB 롤백**: 부분 insert 방지 → 트랜잭션.
- **삭제된 관리자 계정**: `admin` 이 삭제되면 관리자 권한이 없는 상태가 됨 — 시드에서 `admin` DELETE 방지 정책은 **앱 레벨 안전장치 없음**, 운영 규정으로만 관리.

## 방

- **방장이 오프라인 상태에서 방이 비어버림**: `room_members` 가 0 이면 자동 삭제? → **v2.0 은 유지**. 다시 방장이 들어오면 메시지 히스토리 그대로.
- **방 정원 = 현재 인원 경합**: 두 클라이언트가 동시에 마지막 자리 참여 시도 → `INSERT` 후 `SELECT COUNT` 비교에서 한쪽이 초과 감지 → 초과 쪽 `DELETE` rollback + `STATE_CONFLICT`. 단순 구현이라 race 에 완벽하진 않음(허용 가능).
- **핀 메시지가 삭제됨**: `messages.is_deleted=1` 상태의 핀 → 클라이언트에 "(삭제된 메시지)" 로 표시. 자동 unpin 은 v2.1.

## 메시지

- **수정 시한 종료 직전 수정**: 정확히 5분 경계 → `TIMESTAMPDIFF` 로 서버 시각 기준. 클라 시계 불신.
- **삭제된 메시지에 리액션**: server 가 `is_deleted=1` 체크 후 `STATE_CONFLICT`.
- **답장 원본이 다른 방**: INSERT 전에 `reply_to` 의 room_id 일치 확인.
- **WHISPER 대상이 방에 없음**: 여전히 저장(보낸 사람에게만 표시됨). 클라는 "(귓속말) 대상 없음" 힌트 없이 조용히 저장 — v2.1 에서 사용자 경험 개선.

## DM

- **차단 상태에서 DM**: 송신 `BLOCKED`(7) reject. 양방향 차단이면 어느 쪽에서도 불가.
- **상대가 나를 삭제(차단 아님)**: DM 은 여전히 가능. 관계는 "친구 아님".
- **상대가 오프라인**: `DM_SEND_RES|0` 성공. 알림은 상대가 온라인 될 때 `DM_LIST` 로 확인.

## 친구

- **상호 요청**: A→B pending 상태에서 B→A 요청 → `DUPLICATE`(3). B 가 A 의 pending 을 accept 하도록 안내.

## 네트워크

- **부분 패킷 도착**: 수신 버퍼에 쌓다가 `\n` 완성 시 파싱. 4KB 넘도록 `\n` 없으면 drop.
- **`\n` 여러 개 한 번에**: while 루프로 모두 처리.
- **NAT 시간제한**: keepalive 로 회피.
- **소켓 send 도중 close 된 fd**: mutex 없이 send 중이라면 안전(다른 스레드가 해당 fd 에 close+새로 open 한 상태일 수 있음). 이를 피하기 위해 **브로드캐스트는 세션 테이블에서 fd 를 인덱스로 잡고, 세션 제거는 반드시 mutex 보호**.

## UI

- **터미널 크기 < 80×24**: 입력 불가 상태 전환. 리사이즈되면 복귀.
- **UTF-8 폭 계산 실패**: 문자 단위 폭 2/1 매핑 테이블 사용. 예외 문자는 폭 1 로 간주.
- **한 줄 메시지만 허용**: 클라 입력창에서 `\n` 은 전송 트리거. 다중 행 입력은 v2.0 미지원.
