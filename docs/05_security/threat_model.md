# 위협 모델 (STRIDE 간이)

## 1. 자산

- 사용자 자격 증명(비밀번호 해시).
- 메시지 내용(개인 DM 포함).
- 관리자 기능 (Out-of-Scope, 구현 범위 외).

## 2. STRIDE 분석

| 범주 | 위협 | 완화책 | 미완화 잔여 리스크 |
|------|------|--------|-------------------|
| **S**poofing | 타인 ID 로 로그인 | SHA-256 해시 비교, 중복 로그인 차단, 실패 5회 후 close | 평문 전송 → MITM 시 캡처 가능(로컬망 가정) |
| **T**ampering | 패킷 변조로 타인 메시지 전송 | 모든 요청은 세션의 `user_id` 기준으로 처리(페이로드의 from_id 무시) | 소켓 하이재킹(낮음) |
| **R**epudiation | 메시지 부인 | `messages` 에 from_id + created_at 불변 저장 | 감사 로그 별도 없음 |
| **I**nformation disclosure | 타인 DM/오픈채팅 외 방 조회 | 권한 검사, prepared statement, 방 멤버십 검사 | 관리자는 전부 조회 가능 |
| **D**oS | 접속 포화, 대용량 패킷 | `MAX_CLIENTS`, `PKT_MAX=2048`, 로그인 타임아웃 60초, 재시도 5회 | 단일 IP 다중 접속 제한 없음(P4) |
| **E**levation of privilege | 방장 기능 무단 사용 | 모든 핸들러가 세션·DB 기준 권한 재검사 | is_admin 탈취 시 전권(로그인 실패 제한으로 간접 완화) |

## 3. 완화되지 않은 리스크 — 공식 명시

1. **전송 암호화 없음** — 사용은 로컬/신뢰 네트워크로 한정. TLS 도입은 v2.1 이상.
2. **비밀번호에 salt 없음** — rainbow table 취약. v2.1 에서 per-user salt 도입.
3. **속도 제한 없음** — 동일 IP 의 반복 접속·가입 시도 차단 미구현.

## 4. 로깅·모니터링 기본선

- 로그인 실패, 권한 실패, 패킷 파싱 실패는 `[WARN]` 로그.
- 메시지 content 는 로그에 쓰지 않는다(프라이버시).

## 5. 배포 체크리스트

- [ ] 시드 `admin` 계정 비밀번호를 강한 것으로 변경
- [ ] MySQL root 계정이 아닌 전용 사용자로 접속(`chat_app@localhost`, 필요 권한만)
- [ ] 방화벽에서 서버 포트(기본 8080) 외부 노출 제한
- [ ] `sql/schema.sql` 이외의 SQL 파일로 개발 덤프가 유출되지 않게 주의
