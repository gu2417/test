# 목적 및 범위

## 1. 프로젝트 목적

콘솔 환경에서 동작하는 **경량 실시간 채팅 애플리케이션**을 C11 로 구현한다.
카카오톡의 1:1 / 그룹 / 오픈 채팅 개념과 Google Chat 의 Space 개념을
ANSI 기반 TUI 로 제공한다. 모든 영속 데이터는 MySQL 에 저장한다.

## 2. 대상 사용자

- 소규모(≤100 동시 접속) 학습·사내 커뮤니티.
- GUI 가 제한되거나 CPU/메모리가 부족한 **저사양 노트북** 사용자.

## 3. In-Scope (이번 버전에서 만든다)

- TCP Server-Client 분리 구조, 멀티스레드 서버.
- MySQL 영속 저장(유저/친구/방/메시지/리액션/설정).
- 콘솔 TUI: 로그인·메인·채팅·마이페이지·설정 화면.
- 슬래시 커맨드 기반 입력.
- 크로스 플랫폼 빌드: **Linux**, **Windows(MinGW)**, **macOS(Darwin)**.
- 보안 기본선: 비밀번호 SHA-256 해시, SQL prepared statement 강제.

## 4. Out-of-Scope (이번 버전에서 만들지 않는다)

- GUI / 웹 클라이언트.
- TLS 암호화 전송(평문 전송은 로컬 네트워크 한정 — NFR-04 명시).
- 파일/이미지 전송, 음성/화상 통화.
- 푸시 알림(외부 서비스 연동).
- 분산 서버 · 샤딩 · 로드밸런싱.
- 모바일 · 모바일 터미널 최적화.

## 5. 성공 기준(High-Level)

1. Linux/Windows/macOS 각 환경에서 동일 바이너리 동작(동일 소스 기준).
2. 100 동시 접속에서 NFR-02(50ms 이하) 달성(로컬 네트워크).
3. 서버 재시작 후 모든 메시지·유저·방이 유지됨(NFR-08).
4. 비정상 종료 시 서버는 크래시하지 않음(NFR-03).

## 6. requirements.md 와의 관계

이 문서 집합은 `requirements.md` v2.0.0 의 **설계 파생물**이다.
요구사항이 변경되면 `requirements.md` → `docs/` 순으로 갱신한다.
FR/NFR ID 는 그대로 유지한다.
