# 디자인 토큰

## 1. 색상 시스템

### 1.1 시맨틱 롤 (2단계 — 256색 / 8색)

| 롤 | 용도 | 256-color | 8-color fallback |
|----|------|-----------|------------------|
| `bg.base` | 기본 배경 | (터미널 기본) | (터미널 기본) |
| `bg.surface` | 카드·패널 배경 강조 | `\e[48;5;236m` (#303030) | (동일, reset) |
| `bg.elevated` | 선택/호버 | `\e[48;5;238m` | `\e[7m` (inverse) |
| `fg.primary` | 본문 | `\e[38;5;252m` (#d0d0d0) | `\e[37m` |
| `fg.secondary` | 보조 | `\e[38;5;245m` (#8a8a8a) | `\e[90m` |
| `fg.tertiary` | 메타/타임스탬프 | `\e[38;5;240m` | `\e[90m` |
| `accent.primary` | 액센트(링크·포커스) | `\e[38;5;75m` (#5fafff) | `\e[34m` |
| `accent.on` | 액센트 배경 위 텍스트 | `\e[38;5;231m` | `\e[97m` |
| `status.online` | 온라인 | `\e[38;5;42m` (#00d774) | `\e[32m` |
| `status.busy` | 바쁨 | `\e[38;5;214m` (#ffaf00) | `\e[33m` |
| `status.dnd` | 방해금지 | `\e[38;5;203m` (#ff5f5f) | `\e[31m` |
| `status.offline` | 오프라인 | `\e[38;5;240m` | `\e[90m` |
| `feedback.success` | 성공 | `\e[38;5;42m` | `\e[32m` |
| `feedback.warn` | 경고 | `\e[38;5;214m` | `\e[33m` |
| `feedback.error` | 오류 | `\e[38;5;203m` | `\e[31m` |
| `divider` | 구분선 | `\e[38;5;238m` | `\e[90m` |

### 1.2 light 테마

`bg.surface=255`, `fg.primary=235`, `divider=252`. 액센트·상태색은 동일 재사용(대비 충분).

### 1.3 터미널 역량 감지

시작 시 `$COLORTERM`, `$TERM`, `tput colors` 를 확인해:
- `truecolor`/`24bit`/256 지원 → 시맨틱 롤의 256색 값 적용
- 8색만 지원 → fallback 적용
- 무색(`TERM=dumb`) → 모든 색 시퀀스 생략(구조만 출력)

결과는 `theme_caps` 플래그로 **시작 시 한 번** 확정. 런타임 분기 금지.

## 2. 간격 (Spacing)

TUI 는 행·열 단위. "간격" 은 빈 행/공백 열로 표현.

| 토큰 | 값 | 용도 |
|------|----|------|
| `space.0` | 0 | 붙임 |
| `space.1` | 1 공백/행 | 인접 요소 |
| `space.2` | 2 공백/행 | 컴포넌트 내부 |
| `space.3` | 3 공백 | 섹션 간 (수평) |
| `space.section` | 1 공란 행 + 1 희미한 divider | 섹션 간 (수직) |

**원칙**: 구분선보다 **여백**을 먼저 쓴다. 선은 구조상 꼭 필요할 때만.

## 3. 타이포그래피 (텍스트 스타일)

폰트 크기 변경 불가 → **가중치 + 색 + 케이스** 로 위계 표현.

| 레벨 | 스타일 | 예 |
|------|--------|-----|
| `title` | `\e[1m` bold + `fg.primary` | 화면 제목 |
| `h2` | bold + `fg.primary` | 섹션 헤더 |
| `body` | `fg.primary` | 본문 |
| `caption` | `fg.secondary` | 메타 설명 |
| `meta` | `fg.tertiary` | 시간·카운트 |
| `label` | 대문자 + `\e[2m` dim | `ONLINE · 3` 같은 구획 라벨 |
| `link` | `accent.primary` + `\e[4m` underline (옵션) | 선택 가능 항목 강조 |
| `me-action` | `\e[3m` italic + `fg.secondary` | `/me` 액션 |

이탤릭 미지원 터미널(Windows conhost 등): italic → dim 으로 fallback.

## 4. 아이콘 (ASCII-safe Unicode BMP)

Nerd Font 의존 없음. 대부분 BMP · East-Asian Ambiguous 안전 문자.

| 의미 | glyph | 비고 |
|------|-------|------|
| 뒤로 | `←` / `‹` | 헤더 좌측 |
| 설정 | `⚙` | |
| 관리자 | `▤` | 관리자 패널 |
| 메뉴·더보기 | `⋯` | 헤더 우측 |
| 검색 | `⌕` | 입력창 prefix |
| 핀 / 공지 | `◈` | 비밀번호 방도 동일 |
| 첨부(미래) | `+` | |
| 전송 | `⏎` | 입력창 우측 힌트 |
| 읽음 카운터 | 숫자 `3` / `12` / `99+` | accent.primary · bold, `0` 은 공백 |
| 상태 점 | `●` | 색으로 상태 전달 |
| 미읽음 (목록) | `●` (accent) | 왼쪽 원형 (방 목록) |
| 뱃지 원 | `❶❷❸…` or `(3)` fallback | 5까지 기호, 이상은 숫자 |
| 오픈방 | `#` | 해시 |
| DM | `✉` | |
| 귓속말 | `✉` | whisper 메시지 prefix |
| me-action | `✦` | `/me` 액션 prefix |
| 방장 | `★` | 멤버 목록 |
| 공동 방장 | `☆` | 멤버 목록 |
| 체크박스 on | `◉` / off `○` | |
| 토글 on | `●─` / off `─●` | 세그먼트 컨트롤 |
| 트리 화살 | `▸` (닫힘) / `▾` (열림) | 사이드바 그룹 |
| 포커스 커서 | `▌` | 선택 항목 왼쪽 |
| ASCII fallback | `>`, `*`, `o`, `[x]`, `[ ]`, `|`, `+` | 유니코드 미지원 시 |

`LANG` / `LC_ALL` 이 UTF-8 이 아니면 전체 ASCII fallback 셋으로 자동 스위치.

## 5. 박스 드로잉

기본(유니코드):
```
╭─────────╮   ├─────┤   │  │   ·  ·  ·  (얕은 구분)
│         │   ╭─────╮   ╰──╯
╰─────────╯
```

ASCII fallback:
```
+---------+   +-----+   | |    - - -
|         |   +-----+   +-+
+---------+
```

**헤더/입력창/카드만 테두리 사용**. 내부 리스트는 여백 + 얕은 divider(`─── · ───` 또는 3줄 공백 전)로 분리.

## 6. 닉네임 색 (아바타 매칭)

이니셜 아바타 배경색과 닉네임 색은 **같은 팔레트**에서 해시로 결정:

```c
static const int AV[] = { 33, 39, 45, 75, 81, 111, 147, 213, 208, 172 }; /* 256색 index */
uint32_t h = fnv1a(user_id);
int idx = h % (sizeof(AV)/sizeof(AV[0]));
/* bg=AV[idx], fg=231 (거의 흰색) */
```

8색 fallback: `{34,36,35,33,32,31}` 중 해시 선택.

## 7. 포커스 · 선택 표현

| 상태 | 표현 |
|------|------|
| 기본 | 아무 장식 없음 |
| hover (키보드 이동) | 왼쪽 `▌` (accent 색) + `bg.elevated` |
| selected (확정) | 왼쪽 `▌` + bold |
| disabled | `fg.tertiary` + dim |

마우스 없음 → hover = 현재 키보드 커서 위치.

## 8. 컨텐츠 폭 · Truncation 규칙

터미널 너비를 `W` 라 할 때:

| 요소 | 최대 폭 | Truncation |
|------|---------|-----------|
| 사이드바 | 26 고정 | - |
| 메시지 본문 (상대) | `W − 20` | 줄바꿈 |
| 메시지 본문 (나) | `W − 20` | 줄바꿈 |
| 닉네임 | 20자 | `홍길동…` |
| 방 이름 (TopBar) | `W / 3` | `컴공 스터…` |
| 핀 요약 (TopBar 중앙) | `W / 4` | `오늘 자정…` |
| 카드 메시지 프리뷰 | 40자 | `점심 뭐 먹…` |
| 방 주제 (RoomCard) | 50자 | `알고리즘…` |
| 검색 결과 미리보기 | 60자 | `과제 다들…` |
| 답장 인용 프리뷰 | 40자 | `과제 다들…` |

`…` (U+2026 ellipsis) 사용. ASCII fallback: `...` (3자).

## 9. 타이밍 토큰

렌더 tick 기반(별도 타이머 스레드 없음). 모두 입력/소켓 이벤트 처리 루프에서 틱 카운터로 구현.

| 토큰 | 값 | 용도 |
|------|----|------|
| `timing.tick` | 200ms | 기본 렌더 틱 (타이핑 점 애니메이션, 로딩 스피너) |
| `timing.debounce.search` | 300ms | 오픈채팅 검색 디바운스 |
| `timing.debounce.read` | 100ms | 읽음 카운터 재렌더 디바운스 |
| `timing.banner.auto_dismiss` | 5000ms | info/warn 배너 자동 해제 |
| `timing.banner.friend_req` | 10000ms | 친구 요청 배너 자동 해제 |
| `timing.toast` | 2000ms | Toast 자동 해제 |
| `timing.ping_interval` | 30000ms | PING 전송 간격 |
| `timing.ping_timeout` | 60000ms | PONG 미수신 → 연결 끊김 판정 |
| `timing.reconnect_interval` | 5000ms | 재연결 시도 간격 |
| `timing.inline_confirm` | 5000ms | InlineConfirm 자동 취소 |
| `timing.register_redirect` | 1200ms | 회원가입 완료 후 로그인 화면 자동 전환 |
| `timing.typing_expire` | 5000ms | TYPING_STOP 미수신 시 타이핑 표시 소멸 |

## 10. Z-레이어 모델

복수 오버레이가 동시에 활성일 때 우선순위:

| 레이어 | 요소 | 입력 차단 |
|--------|------|----------|
| 4 | 연결 끊김 배너 (urgent) | 차단 안 함 (배너만) |
| 3 | Modal · 리액션 선택기 · 커맨드 자동완성 | 레이어 0 차단 |
| 2 | Banner 알림 스택 | 차단 안 함 |
| 1 | MembersPanel · SearchOverlay | 뒤 화면 포커스 차단 |
| 0 | 화면 콘텐츠 (기본 레이어) | - |

**규칙**: 레이어 3이 활성이면 레이어 0~1의 키 이벤트는 무시. `Esc` 는 항상 현재 최상위 레이어 닫기.

## 11. 금기

- 화려한 풀컬러 그라디언트, 블록 문자 벽 (`█`, `▓` 연속) — 저사양 터미널 렌더 비용 + 가독성 해침
- 섹션마다 서로 다른 배경색 — 두뇌 피로
- 모든 줄에 색 변경 — ANSI escape 이 로그·복붙에 방해
- 느낌표 많은 문구, 이모지 남용 — 서비스 톤 무너짐
- `timing.tick` 보다 짧은 재렌더 루프 — CPU 낭비
