# 인터랙션 패턴

로딩 상태, 빈 상태, 오류, 재연결 등 범용 UX 패턴 정의.
모든 화면에서 일관된 동작을 보장하기 위한 공통 가이드.

---

## 1. 로딩 상태

### 1-1. 인라인 스피너

서버 응답 대기 중 (목록 로드, 히스토리 로드 등):

- `GtkSpinner`를 콘텐츠 영역 중앙에 배치.
- `gtk_spinner_start()` 호출로 애니메이션 시작, 응답 수신 시 `gtk_spinner_stop()` 후 위젯 숨김.

### 1-2. 화면 단위 초기 로딩

화면 진입 직후 첫 데이터 수신 전:

- 메시지 영역에 `GtkSpinner` + `GtkLabel "메시지 로드 중..."` 수직 중앙 배치.
- Composer(`GtkEntry`)는 즉시 활성화(선제 입력 허용).
- 히스토리 수신 완료 시 스피너 제거 후 `GtkListView`로 메시지 목록 채움.

### 1-3. 히스토리 상단 로드

스크롤 최상단에서 과거 메시지 추가 요청 중:

- `GtkScrolledWindow` 상단에 작은 `GtkSpinner` + `GtkLabel "이전 메시지 불러오는 중..."` 삽입.
- 수신 완료 시 스피너 제거 후 메시지를 목록 상단에 prepend (스크롤 위치 유지).

---

## 2. 빈 상태 (Empty State)

빈 상태는 **안내 `GtkLabel` 1개 + 행동 유도 `GtkButton` 1개**. 콘텐츠 영역 수직/수평 중앙 정렬.

### 2-1. 친구 목록 비어있을 때

- `GtkLabel`: "아직 친구가 없습니다."
- `GtkButton "친구 추가"` → 친구 추가 다이얼로그 열기.

### 2-2. 채팅 탭 비어있을 때

- `GtkLabel`: "참여 중인 채팅방이 없습니다."
- `GtkButton "새 채팅방 만들기"` → 방 생성 다이얼로그 열기.

### 2-3. 오픈채팅 탐색 — 검색 결과 없음

- `GtkLabel`: "검색 결과가 없습니다. 다른 키워드로 검색하거나 직접 만들어보세요."
- `GtkButton "+ 오픈채팅 만들기"` → 방 생성 다이얼로그 열기.

### 2-4. 채팅방 — 메시지 없음 (새 방)

- `GtkLabel`: "아직 채팅이 없습니다 — 메시지를 보내보세요."
- Composer가 이미 활성 상태이므로 별도 버튼 불필요.

### 2-5. 친구 요청 없음

- `GtkLabel`: "받은 친구 요청이 없습니다."

### 2-6. 메시지 검색 결과 없음

- `GtkLabel`: "현재 방에서 해당 키워드를 찾을 수 없습니다."

## 3. 오류 상태

### 3-1. 서버 응답 오류 (일시적)

Composer 위에 `GtkRevealer`(슬라이드-인) + `GtkLabel` 표시. 3초 후 자동 숨김:

- 오류 텍스트 예: "서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요."

### 3-2. 클라이언트 유효성 검사 오류

서버에 패킷을 보내지 않고 클라이언트에서 즉시 차단. 동일한 `GtkRevealer` 배너 사용:

- "메시지는 500자를 초과할 수 없습니다. (현재 N자)"
- "/edit: 5분이 지난 메시지는 수정할 수 없습니다."

### 3-3. 권한 오류

- `GtkRevealer` 배너: "방장만 사용할 수 있습니다."

---

## 4. 연결 끊김 & 재연결

### 4-1. 연결 끊김 감지

PONG 미수신 60초 → 모든 화면 상단에 `GtkRevealer`(urgent 배너):

- `GtkLabel`: "서버와의 연결이 끊겼습니다. 재연결 시도 중... (1/3)"
- `GtkButton "종료"`: 클라이언트 종료.
- 배너는 해제 불가(urgent 레벨). Composer는 `gtk_widget_set_sensitive(FALSE)` 처리.

### 4-2. 재연결 시도

3회 시도, 각 5초 간격. `GtkLabel` 카운터 업데이트:

- "재연결 시도 중... (2/3)"

### 4-3. 재연결 성공

`GtkRevealer` 배너 닫기 + 하단 Toast `GtkRevealer`:

- "✓ 다시 연결되었습니다"

현재 화면 자동 복구: 채팅방이면 `HISTORY_REQ` 재발송 → 끊긴 동안 놓친 메시지 수신.

### 4-4. 재연결 실패 (3회 모두)

배너 `GtkLabel` 변경:

- "연결에 실패했습니다."
- `GtkButton "재시도"`: 3회 카운터 리셋 후 재시작.
- `GtkButton "종료"`: 클라이언트 종료.

---

## 5. 친구 요청 수신 플로우

### 5-1. 실시간 배너 (어느 화면에서나)

`GtkRevealer`(슬라이드-인) + `GtkLabel` + `GtkButton` 조합:

- `GtkLabel`: "홍길동 님이 친구 요청을 보냈습니다."
- `GtkButton "수락"` → `FRIEND_ACCEPT` 전송. Toast "✓ 홍길동 님과 친구가 되었습니다".
- `GtkButton "거절"` → `FRIEND_REJECT` 전송. 배너 소멸.
- `GtkButton "×"` → 보류. 메인 친구 탭 뱃지 카운트 증가.
- 10초 후 자동 해제.

### 5-2. 보류된 요청 목록 (친구 탭 상단)

`GtkListBox`로 미처리 요청 표시. 각 `GtkListBoxRow`:

- `GtkLabel` 닉네임, ID, 상태 메시지.
- `GtkButton "수락"` + `GtkButton "거절"`.

---

## 6. 컨텍스트 메뉴 (메시지 우클릭)

메시지 위젯 우클릭 → `GtkPopoverMenu` 표시:

| 메뉴 항목 | 동작 |
|-----------|------|
| 답장 | 답장 미리보기 `GtkRevealer` 열기 |
| 복사 | 클립보드에 메시지 내용 복사 |
| 수정 | Composer에 수정 대상 메시지 내용 로드 (본인 메시지, 5분 이내만) |
| 삭제 | `GtkAlertDialog` 확인 후 `MSG_DELETE` 전송 |
| 귓속말 | Composer에 `/w <user_id>` 삽입 |

권한이 없는 항목은 `gtk_widget_set_sensitive(FALSE)`로 비활성화.

---

## 7. 답장 미리보기

답장 선택 시 `GtkRevealer`가 Composer(`GtkEntry`) 위에 슬라이드-인:

- `GtkLabel` 인용 텍스트: 최대 40자, 초과 시 `…` truncate.
- `GtkLabel` 원본 발신자 닉네임.
- `GtkButton "×"`: 답장 취소, `GtkRevealer` 닫기.
- Composer에서 Enter 입력 시 `MSG_REPLY` 패킷 발송.

---

## 8. 모달 우선순위 및 Z-레이어

복수의 이벤트가 동시에 발생할 때 우선순위:

| Z-레이어 | 요소 | 비고 |
|----------|------|------|
| 4 (최상) | 연결 끊김 배너 (`GtkRevealer`) | urgent, 해제 불가 |
| 3 | `GtkAlertDialog` (확인 창) | Escape로 닫힘 |
| 3 | `GtkPopoverMenu` (컨텍스트 메뉴) | Escape로 닫힘 |
| 2 | 알림 배너 스택 (`GtkRevealer`) | 최대 3개 |
| 1 | Toast 스낵바 (`GtkRevealer`) | 하단 중앙 |
| 0 | 화면 콘텐츠 | 기본 레이어 |

**규칙**: 레이어 3 이상이 활성이면 레이어 0의 입력 이벤트는 무시.

---

## 9. 메시지 전송 실패

네트워크 오류로 메시지 전송 실패 시:

- 메시지 행 우측에 `GtkLabel "✗ 전송 실패"` 인라인 표시.
- `GtkButton "재시도"` 제공 → 재전송 시도.
- 재전송 성공 시 실패 표시 제거 + 정상 렌더.

---

## 10. 중복 로그인 처리

다른 기기에서 동일 계정 로그인 시 현재 클라이언트에게 `GtkAlertDialog`:

- 제목: "다른 기기에서 로그인"
- 본문: "다른 기기에서 이 계정으로 로그인했습니다. 현재 세션이 종료됩니다."
- `GtkButton "확인"` 또는 5초 후 자동으로 LOGIN 화면 전환.
