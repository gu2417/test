# 인터랙션 패턴

로딩 상태, 빈 상태, 오류, 재연결, 커맨드 자동완성 등 범용 UX 패턴 정의.
모든 화면에서 일관된 동작을 보장하기 위한 공통 가이드.

---

## 1. 로딩 상태

### 1-1. 인라인 스피너

서버 응답 대기 중 (목록 로드, 히스토리 로드 등):

```
   ╭──────────────────────────────────────────────────╮
   │  · · ·  불러오는 중                              │
   ╰──────────────────────────────────────────────────╯
```

- 점 3개 `· · ·` 를 순환 애니메이션 (`·   `, `· · `, `· · ·`) — **렌더 틱 기반**, 별도 스레드 없음.
- 200ms 마다 틱 전진. 타이머는 알림 TTL 처리와 동일한 루프에서 처리.

### 1-2. 화면 단위 초기 로딩

화면 진입 직후 첫 데이터 수신 전:

```
╭──────────────────────────────────────────────────────────────────────────────────────╮
│ ←  # 컴공 스터디그룹                                                                  │
╰──────────────────────────────────────────────────────────────────────────────────────╯

                                · · ·  메시지 로드 중

╭──────────────────────────────────────────────────────────────────────────────────────╮
│ ▌ 메시지 입력                                                /help         ⏎ 전송    │
╰──────────────────────────────────────────────────────────────────────────────────────╯
```

Composer 는 즉시 활성화(선제 입력 허용). 히스토리 수신 완료 시 메시지 뷰 채움.

### 1-3. 히스토리 상단 로드 (`Ctrl+U`)

스크롤 최상단에서 과거 메시지 추가 요청 중:

```
              ↑  · · ·  이전 메시지 불러오는 중                ●  3 새 메시지
```

수신 완료 시 기존 메시지 위에 자연스럽게 prepend (스크롤 위치 유지).

## 2. 빈 상태 (Empty State)

빈 상태는 **안내 문구 1줄 + 행동 유도 힌트 1줄**. 장식 최소화.

### 2-1. 친구 목록 비어있을 때

```
   ONLINE · 0

                     아직 친구가 없습니다.
                     /friend add <ID> 로 첫 친구를 추가해보세요.
```

### 2-2. 채팅 탭 비어있을 때

```
                     참여 중인 채팅방이 없습니다.
                     /room create <이름> 또는 /join <room_id> 로 시작하세요.
```

### 2-3. 오픈채팅 탐색 — 검색 결과 없음

```
   ⌕  스터디zzz

   검색 결과 없음
   "스터디zzz" 와 일치하는 방이 없습니다.
   다른 키워드로 검색하거나 직접 만들어보세요.  + 오픈채팅 만들기
```

### 2-4. 채팅방 — 메시지 없음 (새 방)

```
                     아직 메시지가 없습니다.
                     첫 메시지를 보내보세요!
```

### 2-5. 친구 요청 없음

```
   받은 요청 · 0

                     받은 친구 요청이 없습니다.
```

### 2-6. 메시지 검색 결과 없음

```
   ⌕  검색어: "없는단어"

                     현재 방에서 "없는단어" 를 찾을 수 없습니다.
```

## 3. 오류 상태

### 3-1. 서버 응답 오류 (일시적)

Composer 위 1행에 인라인 표시. 3초 후 자동 사라짐:

```
  ✗ 서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요.     ← feedback.error
```

### 3-2. 클라이언트 유효성 검사 오류

커맨드 파싱 실패 또는 범위 초과:

```
  ✗ /edit <msg_id> <내용> 형식으로 입력하세요.
  ✗ 메시지는 500자를 초과할 수 없습니다. (현재 512자)
  ✗ /edit: 5분이 지난 메시지는 수정할 수 없습니다.
```

서버에 패킷을 보내지 않고 클라이언트에서 즉시 차단.

### 3-3. 권한 오류

```
  ✗ 방장 또는 관리자만 사용할 수 있습니다.    ← feedback.error
```

## 4. 연결 끊김 & 재연결

### 4-1. 연결 끊김 감지

PONG 미수신 60초 → 모든 화면 최상단에 Banner 레벨 2/urgent:

```
 ⦿  서버와의 연결이 끊겼습니다. 재연결 시도 중... (1/3)   × 종료
```

- `×` 클릭 = 클라이언트 종료.
- 배너는 해제 불가 (urgent 레벨).
- 이 동안 Composer 는 `disabled` 상태: 입력 가능하지만 전송 시 `✗ 오프라인 상태입니다.` 표시.

### 4-2. 재연결 시도

3회 시도, 각 5초 간격. 배너 카운터 업데이트:

```
 ⦿  재연결 시도 중... (2/3)    마지막 시도  14:08:12
```

### 4-3. 재연결 성공

배너 제거 + Toast:

```
                  ✓ 다시 연결되었습니다
```

현재 화면 상태 자동 복구: 채팅방이면 `HISTORY_REQ` 재발송 → 끊긴 동안 놓친 메시지 수신.

### 4-4. 재연결 실패 (3회 모두)

```
 ⦿  연결에 실패했습니다.    [ 재시도 ]   [ 종료 ]
```

`[ 재시도 ]` = 3회 카운터 리셋 후 재시작. `[ 종료 ]` = 클라이언트 종료.

## 5. 커맨드 자동완성

`/` 입력 직후 커맨드 목록 팝업 (Composer 바로 위).

### 5-1. 커맨드 목록 팝업

```
  ╭──────────────────────────────────────────────────────────────╮
  │  /del    <msg_id>          내 메시지 삭제                    │
  │  /dnd                      방해금지 모드 토글                │
  │  /dm     <id>              1:1 DM 시작           ← 현재 선택 │
  ╰──────────────────────────────────────────────────────────────╯
╭──────────────────────────────────────────────────────────────────────────────────────╮
│ ⌘ /d▌                                                        /help        ⏎ 전송   │
╰──────────────────────────────────────────────────────────────────────────────────────╯
```

- 팝업은 Composer 바로 위에 **오버레이** (z-layer 3). 최대 5개 항목 표시, 스크롤 가능.
- 현재 화면에서 사용 불가능한 커맨드는 `fg.tertiary` + dim 처리.
- 포커스된 항목: 왼쪽 `▌` + `bg.elevated`.

### 5-2. 인자 힌트

커맨드 확정 후 인자 입력 단계:

```
  ╭──────────────────────────────────────────────────────────────╮
  │  /reply <msg_id> <내용>    ← 현재 입력 중인 인자 표시       │
  ╰──────────────────────────────────────────────────────────────╯
╭──────────────────────────────────────────────────────────────────────────────────────╮
│ ⌘ /reply ▌                                                  /help        ⏎ 전송    │
╰──────────────────────────────────────────────────────────────────────────────────────╯
```

### 5-3. 닉네임/ID 자동완성

`/w <tab>`, `/invite <tab>`, `@<tab>` 입력 시 현재 방 멤버 목록 제안:

```
  ╭──────────────────────────────────────────────────╮
  │  홍길동   hong123   ● online                     │
  │  김철수   kim456    ● online        ← 선택        │
  │  이영희   lee789    ○ offline                     │
  ╰──────────────────────────────────────────────────╯
╭──────────────────────────────────────────────────────────────────────────────────────╮
│ ⌘ /w 김▌                                                    /help        ⏎ 전송    │
╰──────────────────────────────────────────────────────────────────────────────────────╯
```

### 5-4. 키 바인딩 (자동완성 팝업 활성 시)

| 키 | 동작 |
|----|------|
| `↑` `↓` | 항목 이동 |
| `Tab` `Enter` | 항목 확정, 팝업 닫기 |
| `Esc` | 팝업 닫기, Composer 입력 유지 |
| 계속 타이핑 | 실시간 필터링 |

## 6. 친구 요청 수신 플로우

### 6-1. 실시간 배너 (어느 화면에서나)

```
 ⦿  홍길동 님이 친구 요청을 보냈습니다.   [1 수락]  [2 거절]   ×
```

- Banner 레벨 0/info. 10초 후 자동 해제.
- `[1 수락]` → `FRIEND_ACCEPT` 즉시 전송. Toast `✓ 홍길동 님과 친구가 되었습니다`.
- `[2 거절]` → `FRIEND_REJECT` 전송. 배너 소멸.
- `×` → 나중에 처리(보류). 메인 `친구` 탭 뱃지에 카운트 증가.

### 6-2. 보류된 요청 목록 (친구 탭 상단)

메인 화면 친구 탭에서 미처리 요청이 있을 때:

```
   받은 요청 · 2

   ╭──────────────────────────────────────────────────────────────╮
   │ ╭──╮                                                         │
   │ │홍│  홍길동   hong123                             [수락] [거절] │
   │ ╰──╯  오늘도 화이팅                                          │
   ╰──────────────────────────────────────────────────────────────╯

   ╭──────────────────────────────────────────────────────────────╮
   │ ╭──╮                                                         │
   │ │박│  박민준   park000                             [수락] [거절] │
   │ ╰──╯  (상태 메시지 없음)                                     │
   ╰──────────────────────────────────────────────────────────────╯

   ─────────────────  ·  ─────────────────

   ONLINE · 3
```

## 7. 모달 우선순위 및 Z-레이어

복수의 이벤트가 동시에 발생할 때 우선순위:

| Z-레이어 | 요소 | 비고 |
|----------|------|------|
| 4 (최상) | 연결 끊김 배너 | urgent, 해제 불가 |
| 3 | Modal (확인 창) | Esc 로 닫힘 |
| 3 | 커맨드 자동완성 팝업 | Esc 로 닫힘 |
| 2 | Banner 알림 스택 | 최대 3개 |
| 1 | Toast 스낵바 | 하단 중앙 |
| 0 | 화면 콘텐츠 | 기본 레이어 |

**규칙**: 레이어 3 이상이 활성이면 레이어 0의 키 이벤트는 무시.

## 8. 메시지 전송 실패

네트워크 오류로 메시지 전송 실패 시:

```
   [14:08]  내 메시지   ✗ 전송 실패   ← feedback.error, 작게
```

- 실패 표시는 메시지 행 우측에 인라인으로 붙음.
- `r` 키 또는 `Enter` → 재전송 시도.
- 재전송 성공 시 `✗ 전송 실패` 제거 + 정상 렌더.

## 9. 중복 로그인 처리

다른 기기에서 동일 계정 로그인 시 현재 클라이언트에게:

```
╭─ 다른 기기에서 로그인 ──────────────────────────╮
│                                                  │
│   다른 기기에서 이 계정으로 로그인했습니다.     │
│   현재 세션이 종료됩니다.                       │
│                                                  │
│                        [ 확인 ]                  │
╰───────────────────────────────────────────────────╯
```

확인 또는 5초 후 자동으로 LOGIN 화면으로 돌아감.
