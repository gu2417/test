# 저사양 렌더링 전략

저사양 Windows/Mac 노트북에서 부드럽게 동작하는 것이 목표. 다음 원칙을 **강제**.

## 1. 부분 갱신(Partial Redraw)

- 각 스크린은 진입 시 1회 `render_full()`, 이후 이벤트 단위 `render_patch(event)`.
- 메시지 추가 = 메시지 뷰 영역에 **1줄만 추가** (스크롤 영역은 `\e[<top>;<bot>r` DECSTBM 사용).
- 상태/탭 전환 시에만 전체 재그리기.

## 2. 터미널 크기

- 최소 80×24. 더 작으면 진입 시 경고 출력 후 동작 제한 모드(요약 와이어프레임).
- `SIGWINCH` 수신 시 크기 재조회 → `render_full`.

## 3. ANSI 최소 사용

- 색상 전환은 메시지당 최대 2~3개 시퀀스. 모든 셀마다 색 변경 금지.
- `clear screen(\e[2J)` 은 전체 재그리기 시에만.
- 256색 시퀀스(`\e[38;5;N m`)는 현대 디자인 토큰용. 터미널 역량 감지 실패 시 8색 fallback 으로 통일 — `design_tokens.md` 의 롤 매핑을 `theme_caps` 플래그로 **시작 시 한 번** 확정, 런타임 분기 금지.
- 유니코드 박스(`╭╮╰╯`)와 아이콘은 `LC_ALL` / `LANG` 이 UTF-8 일 때만. 아니면 `+-|` ASCII 셋 자동 적용.

## 4. 입력

- raw-mode 에서 바이트 단위 `read`. 폴링 루프 금지 — 블로킹 `read` + `select`(또는 `WaitForMultipleObjects`) 로 입력/이벤트 큐 둘 중 먼저 도착한 것을 처리.

## 5. 메모리

| 자원 | 한도 |
|------|------|
| 메시지 링버퍼 | 방당 200개 × 500자 ≈ 100KB |
| 친구 목록 | 최대 500 × 100B ≈ 50KB |
| 방 목록 | 최대 50 × 200B ≈ 10KB |
| 알림 큐 | 8개 고정 |

전체 ≤ 20MB 상주 목표. 동적 할당은 초기화 시점에만 집중.

## 6. CPU

- 유휴 상태에서 CPU 0% 에 가깝게. `sleep`/`select` 로 완전 블로킹.
- 타이머 기반 기능(알림 TTL, 타이핑 인디케이터)은 별도 스레드 없이 **렌더 tick** 에서 처리.

## 7. 네트워크

- 패킷 합성/배치 없이 1 이벤트 = 1 패킷. 2048B 이내 보장.
- PING: 30초 간격. PONG 미수신 60초 → 연결 끊김으로 간주 → 재연결 시도 3회.
