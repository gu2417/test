# 관리자 패널

`is_admin=1` 유저에게만 노출. `/admin` 커맨드 또는 MAIN 화면 사이드바 하단 `⚙ 관리자` 버튼으로 진입.

---

## 1. 진입 경로

```
MAIN 사이드바 (admin 전용):
  +  새 채팅
  ⚙  설정
  ▤  관리자 패널    ← admin 계정일 때만 표시
```

또는 모든 화면에서 `/admin <sub> <args>` 직접 입력.

---

## 3. 유저 목록 탭

`[유저 목록]` 선택 또는 `/admin user_list`:

```
   유저 목록   1,432 명                                        ⌕ 검색
   ────

    상태      ID             닉네임          가입일          마지막 접속
   ──────────────────────────────────────────────────────────────────────
    ● online  hong123        홍길동          2026. 1. 15     방금 전
    ● online  kim456         김철수          2026. 2. 03     방금 전
    ○ offline lee789         이영희          2026. 3. 10     3시간 전
    ○ offline park000        박민준          2025.12. 01     2일 전

   ──────────────────────────────────────────────────────────────────────
   행 선택: Enter  강제 로그아웃: K  계정 정보: I
```

**행 선택 후 컨텍스트 액션:**

```
╭─ 유저 액션 ────────────────────────────────────╮
│   kim456  · 김철수  ● online                   │
│                                                │
│   1. 상세 정보 보기                            │
│   2. 강제 로그아웃                             │
│   0. 취소                                      │
╰────────────────────────────────────────────────╯
```

**강제 로그아웃 확인:**

```
╭─ 강제 로그아웃 확인 ────────────────────────────╮
│                                                  │
│   kim456 (김철수) 를 강제 로그아웃하시겠어요?   │
│   접속이 즉시 종료되며, 상대방에게 알림이 갑니다.│
│                                                  │
│              [ 취소 ]   [ 강제 로그아웃 ]        │
╰───────────────────────────────────────────────────╯
```

성공 시 Toast `✓ kim456 강제 로그아웃 완료`. 행에서 `● online` → `○ offline` 으로 즉시 전환.

## 4. 채팅방 목록 탭

`[채팅방 목록]` 또는 `/admin room_list` (확장 커맨드):

```
   채팅방 목록   7 개 활성
   ────

    유형    ID    방이름               인원    방장         생성일
   ────────────────────────────────────────────────────────────────
    그룹      3   컴공 스터디그룹     12/30   hong123     2026. 1.20
    오픈      5   개발 스터디          8/30   kim456      2026. 2.01
    그룹      8   점심 모임            5/10   lee789      2026. 4.10
    오픈     12   점심 뭐 먹을까      24/50   park000     2026. 3.15

   ────────────────────────────────────────────────────────────────
   행 선택: Enter  강제 삭제: D
```

**채팅방 강제 삭제 확인:**

```
╭─ 채팅방 강제 삭제 ──────────────────────────────╮
│                                                  │
│   "컴공 스터디그룹" (ID:3) 을 강제 삭제합니다.  │
│   현재 참여자 12명이 즉시 추방됩니다.           │
│   이 작업은 되돌릴 수 없습니다.                 │
│                                                  │
│              [ 취소 ]   [ 삭제 ]                 │
╰───────────────────────────────────────────────────╯
```

성공 시 Toast `✓ 채팅방 "컴공 스터디그룹" 삭제됨`. 행 제거 + 참여자 전원에게 시스템 메시지 브로드캐스트.

## 5. 전체 공지 탭

`[전체 공지]` 또는 `/admin broadcast <내용>`:

```
   전체 공지
   ─────────────────  ·  ─────────────────

   현재 접속 중인 12명 전원에게 공지를 전송합니다.

   ╭────────────────────────────────────────────────────────────────╮
   │                                                                │
   │  공지 내용을 입력하세요. (최대 200자)                          │
   │                                                                │
   │  ┌──────────────────────────────────────────────────────────┐ │
   │  │ ▌                                                        │ │
   │  │                                                          │ │
   │  └──────────────────────────────────────────────────────────┘ │
   │                                                                │
   │                            [ 취소 ]   [ 전송 ]                │
   ╰────────────────────────────────────────────────────────────────╯
```

**전송 후 확인:**

```
╭─ 공지 전송 확인 ────────────────────────────────╮
│                                                  │
│  다음 내용을 12명에게 브로드캐스트합니다:        │
│                                                  │
│  "오늘 오후 6시에 서버 점검이 예정되어 있습니다" │
│                                                  │
│              [ 취소 ]   [ 전송 ]                 │
╰───────────────────────────────────────────────────╯
```

전송 성공 시: Toast `✓ 공지 전송 완료 (12명)`.  
수신 측: 모든 화면의 Banner에 `⦿ [서버 공지] 오늘 오후 6시에 서버 점검...` 레벨 2/urgent 로 표시 → 자동 해제 없음.

## 6. 슬래시 커맨드로 직접 실행

관리자 패널 화면 없이도 모든 화면에서 사용 가능:

| 커맨드 | 동작 | 응답 |
|--------|------|------|
| `/admin broadcast <내용>` | 전체 공지 확인 모달 → 전송 | Toast `✓ 전송 완료 (N명)` |
| `/admin kick_user <id>` | 강제 로그아웃 확인 모달 | Toast `✓ 로그아웃 완료` 또는 `⚠ 유저 없음` |
| `/admin stat` | 서버 현황을 Composer 위 인라인 표시 | 메시지 박스 형태 |
| `/admin user_list` | 관리자 패널 유저 목록 탭으로 이동 | 화면 전환 |
| `/admin delete_room <id>` | 방 삭제 확인 모달 | Toast |

`/admin stat` 인라인 응답 예시:

```
   ┌─ 서버 현황 ─────────────────────────────────────────────────────┐
   │  접속  12/100    방  7    메시지 89,241    업타임 3일 14시간    │
   └──────────────────────────────────────────────────────────────────┘
```

---

## 7. 권한 오류

admin 아닌 유저가 `/admin` 커맨드 실행 시:

```
  ✗ 관리자 권한이 필요합니다.     ← Composer 위 1행, feedback.error
```

---

## 8. 키 바인딩

| 키 | 동작 |
|----|------|
| `1` `2` `3` | 탭 전환 (서버현황 / 유저목록 / 채팅방 / 공지) |
| `↑` `↓` | 목록 이동 |
| `Enter` | 항목 선택 / 액션 |
| `K` | 강제 로그아웃 (유저 목록 탭) |
| `D` | 방 삭제 (채팅방 목록 탭) |
| `r` | 목록 새로고침 (`stat` 재요청) |
| `Esc` | MAIN 복귀 |
