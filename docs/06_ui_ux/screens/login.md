# 로그인 · 회원가입

Google Workspace 로그인 페이지의 **중앙 카드** 패턴을 TUI 로 옮김.
단 하나의 초점(입력 카드) + 서비스 마크 + 하단 보조 링크.

## 와이어프레임 — 진입 화면 (100×30 기준)

```


                         ◆  C Chat
                         ─────────────
                         가볍고 빠른 터미널 메신저


             ╭────────────────────────────────────────╮
             │                                        │
             │   계속하려면 로그인하세요              │
             │                                        │
             │   ID        ┌──────────────────────┐   │
             │             │ ▌                    │   │
             │             └──────────────────────┘   │
             │                                        │
             │   비밀번호  ┌──────────────────────┐   │
             │             │  ••••••              │   │
             │             └──────────────────────┘   │
             │                                        │
             │                                        │
             │                   [ 로그인 ]           │
             │                                        │
             ╰────────────────────────────────────────╯

                계정이 없으신가요?  회원가입 →    종료 (q)

```

- 카드 폭 44, 가운데 정렬. 화면이 좁으면 폭 자동 축소(최소 36).
- 상단 브랜드(`◆ C Chat`) · tagline 은 `fg.secondary`.
- 입력 필드는 얇은 박스, 활성 필드는 왼쪽 `▌` 액센트 + 박스 선 색을 `accent.primary`.

## 와이어프레임 — 회원가입

```
             ╭────────────────────────────────────────╮
             │   회원가입                             │
             │   ─────                                │
             │                                        │
             │   ID          [                    ]   │
             │   3~20자 · 영숫자/_                    │
             │                                        │
             │   비밀번호    [                    ]   │
             │   4~30자                               │
             │                                        │
             │   비밀번호 확인[                   ]   │
             │                                        │
             │   닉네임      [                    ]   │
             │   1~20자                               │
             │                                        │
             │   상태 메시지 [                    ]   │
             │   선택 · 0~100자                       │
             │                                        │
             │                   [ 가입하기 ]         │
             ╰────────────────────────────────────────╯

                이미 계정이 있나요?  로그인 ←
```

- 각 필드 바로 아래 1행에 `caption` 으로 규칙 힌트. 유효성 위반 시 같은 자리에서 `feedback.error` 로 메시지 교체.
- 버튼은 Chip 스타일. 포커스 시 `bg=accent.primary` `fg=accent.on`.

## 입력 · 포커스 · 키

| 키 | 동작 |
|----|------|
| `Tab` / `↓` | 다음 필드 |
| `Shift+Tab` / `↑` | 이전 필드 |
| `Enter` | 마지막 필드에서 submit, 그 외는 다음 필드 |
| `Esc` | 진입 화면으로 복귀 / 종료 확인 |
| `Ctrl+U` | 현재 필드 지우기 |

- 비밀번호는 `•` 로 마스킹, 에코 금지.
- 클라 사전 검증 통과 후에만 `LOGIN`/`REGISTER` 송신.

## 상태 피드백

서버 응답(`08_api/error_codes.md`) 매핑 — 카드 바로 아래 1행:

| code | 문구 (피드백 레벨) |
|-----:|---------------------|
| 0 | `✓ 로그인 완료` (success) → MAIN 전환 |
| 1 | `입력을 확인해 주세요` (error) |
| 2 | `ID 또는 비밀번호가 맞지 않습니다` (error) |
| 3 | `이미 존재하는 ID 입니다` (error) |
| 6 | `이미 접속 중인 계정입니다` (warn) |
| 9 | `일시적인 서버 오류` (error) |

회원가입 성공:
```
                   ✓ 가입이 완료되었습니다
                        잠시 후 로그인으로 돌아갑니다
```
1.2초 후 자동 전환.

## 브랜드 · 비주얼

- 로고는 텍스트 `◆ C Chat`. 색은 `accent.primary`, **절대 블록 아스키 아트 쓰지 않음**.
- 카드 안쪽 패딩은 `space.3` (양옆 3칸 · 위아래 1행).
- 8색 터미널에서는 카드 테두리·활성 박스만 `blue` 로, 나머지는 리셋 — 단순화.
