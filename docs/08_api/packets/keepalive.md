# keepalive 패킷

## 목적

- NAT/방화벽 idle-timeout 회피.
- 상대 종료(RST 미도달) 감지.

## 흐름

```
PING
PONG
```
- 클라가 30초마다 `PING` 전송.
- 서버는 즉시 `PONG` 응답.
- 서버가 90초 동안 어떤 패킷(포함 PING)도 받지 못하면 해당 세션 종료.
- 클라가 `PONG` 을 60초 동안 못 받으면 재연결 루프 진입.

## 포맷

필드 없음. `PING\n`, `PONG\n`.

## 비용

- 연결당 30초에 1회, 고정 4byte 송·수신. 100 동시 연결 기준 초당 ~13 패킷. 무시 가능.
