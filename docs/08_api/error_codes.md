# 에러 코드

`*_RES` 응답의 첫 번째 필드로 사용. `0 = OK`, 이외는 실패.

| 코드 | 이름 | 의미 | 대표 상황 |
|-----:|------|------|-----------|
| 0 | `OK` | 성공 | 정상 처리 |
| 1 | `INVALID_INPUT` | 입력값 규칙 위반 | 길이 초과, 허용 외 문자 |
| 2 | `AUTH_FAIL` | 인증 실패 | 비밀번호 불일치, 미존재 아이디 |
| 3 | `DUPLICATE` | 중복 | 이미 존재하는 아이디/닉/친구/리액션 |
| 4 | `NOT_FOUND` | 대상 없음 | 유저·방·메시지 id 미존재 |
| 5 | `FORBIDDEN` | 권한 부족 | 방장/관리자 전용 작업 |
| 6 | `STATE_CONFLICT` | 상태 불일치 | 이미 삭제된 메시지, 정원 초과, 수정 시한 초과 |
| 7 | `BLOCKED` | 차단 관계 | 차단된 사용자에게 DM/친구요청 |
| 8 | `RATE_LIMIT` | 속도 제한 | 스팸 방지(v2.1에서 구체화) |
| 9 | `SERVER_ERROR` | 서버 내부 오류 | DB 오류, 예외 |
| 10 | `PROTOCOL_ERROR` | 프로토콜 위반 | 필드 수 부족, 잘못된 TYPE |

## 응답 페이로드

- 성공 시: `RES|0|<데이터...>`
- 실패 시: `RES|<code>` 만. 사람이 읽는 메시지는 **붙이지 않음**(클라가 code 기반 현지화).

예:
```
LOGIN_RES|0|alice|앨리스
LOGIN_RES|2
ROOM_JOIN_RES|6        # 정원 초과
MSG_EDIT_RES|5         # 권한 부족
```

## 클라이언트 메시지 매핑 예

| code | UI 표시 |
|------|---------|
| 1 | "입력이 올바르지 않습니다." |
| 2 | "아이디 또는 비밀번호가 잘못되었습니다." |
| 3 | "이미 존재합니다." |
| 4 | "대상을 찾을 수 없습니다." |
| 5 | "권한이 없습니다." |
| 6 | "현재 상태에서는 할 수 없습니다." |
| 7 | "차단된 상대입니다." |
| 8 | "요청이 너무 잦습니다. 잠시 후 시도하세요." |
| 9 | "서버 오류가 발생했습니다." |
| 10 | "프로토콜 오류(내부)." |
